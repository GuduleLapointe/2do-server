2do directory

2do.directory is a public, hypergrid search engine for OpenSimulator. It provides results for in-world search places, land for sale, classifieds and events. Data are collected from several OpenSimulator grids.

This page gives grid owners instructions to enable in-world search and events on their own grid.

Enable in-world search in your grid

To enable in-world search in your grid without installing the web application:

  1. Download OpenSimSearch.Module.dll and put it inside your OpenSim bin/ folder.

  2. For each simulator, add the following settings in OpenSim.ini file (even for grids, this config goes in regions OpenSim.ini, not in Robust.ini). Don't forget to replace Your Grid and yourgrid.org:8002 with your own values).

    [Search]
  ;; Set this section to enable in-world search
  SearchURL = "http://2do.directory/helpers/query.php?gk=http://yourgrid.org:8002"
  ;; the gk parameter is needed to avoid hypergrid issues and must be set to your grid Gatekeeper URL

[DataSnapshot]
  ;; Set this section to include your grid places, land for sale and classifieds in search results
  index_sims = true
  gridname = "Your Grid"
  snapshot_cache_directory = "./DataSnapshot"
  DATA_SRV_2do = http://2do.directory/helpers/register.php

  ;; You can register to multiple search engines:
  ;DATA_SRV_MISearch = http://metaverseink.com/cgi-bin/register.py
  ;DATA_SRV_OtherEngine = http://example.org/helpers/register.py

  3. Restart all simulators.

Include your grid events in 2do.directory

Please send a mail to dev@2do.pm, including the public URL of your calendar and your grid default landing area (which will be used when the destination an event is not set or not properly formatted).

The easiest format is iCal standard (.ics), so any calendar service allowing an ics URL shoud do the trick (including Google Agenda).

  • Times must be set correctly according to your calendar time zone. 2do engine will take care of the conversion to grid time (PST/PDT).
    Example: If your events is at noon grid-time (00:00 PM PST/PDT) but your own calendar is on Europe/Paris time zone, set your event the local time in Paris (20:00 or 21:00 depending on daylight saving).
  • Think about daylight saving time, particularly for recurring events. Daylight Saving Time follow differrent rules depending on countries, even the date of the switch varies. So a given time in your local area will not always correspond to the same grid time (unless you live in Los Angeles of course). E.g.:
    • If you your recurring events happens at noon, grid-time, whatever day of the year, you should set your own calendar timezone to PST/PDT (and your local time might vary).
    • If you want your local time to remain unchanged, set your calendar to your local time zone (and the grid-time might vary).
  • Location should be set to the teleport URL of the destination, e.g. yougrid.org:8002:Welcome Area
  • Category can be set different ways:
    • If your calendar handles categories, use this feature.
    • Otherwisse, you can insert a tag in the description, in the form: [[Category:Music]]
    • If none of the above apply, 2do engine will try to guess from the event title and description.

    Current categories are inherited from original HYPEvents project
    Art, Education, Fair, Lecture, Literature, Music, Roleplay, Social

    Work is in progress to convert them to official viewer categories (Discussion, Sports, Live DJ, Live Music, Commercial, Nightlife/Entertainment, Games/Contests, Pageants, Education, Arts and Culture, Charity/Support Groups).

Other formats are possible, but they would require an analysis and the development of a specific parser.

2do.directory is an implementation of Speculoos' 2do HYPEvents and Flexible helper scripts.
These helpers are also available as a WordPress plugin w4os Web interface for OpenSimulator.
